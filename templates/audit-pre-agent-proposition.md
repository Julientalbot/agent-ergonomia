## proposition courte

**Mission : vérifier votre premier cas IA**

Client : [organisation]
Interlocuteur : [nom / fonction]
Date : [date]
Situation étudiée : [ex. demandes clients, reporting, suivi administratif, préparation de dossiers]
Équipe concernée : [service / métier / nombre de personnes]

### pourquoi cette mission

Vous envisagez d’utiliser l’IA pour [objectif exprimé par le client].

Avant de choisir un outil ou de lancer un test, il faut vérifier ce qui se passe réellement dans le travail :

- quelles tâches prennent du temps ;
- quelles décisions demandent du jugement ;
- où l’information est fiable ou non ;
- ce qui est déjà vérifié, corrigé ou contourné par les équipes ;
- ce que l’IA pourrait simplifier ;
- ce qu’il pourrait au contraire compliquer.

Cette mission sert à distinguer les bons cas d’usage des faux bons cas d’usage.

### objectif

À l’issue de la mission, vous disposez d’un avis clair sur :

- où l’IA peut créer une aide réelle ;
- où il risque de produire du travail caché ;
- quelles conditions minimales doivent être réunies avant de tester ou déployer.

La mission ne vend pas une solution IA et ne promet pas un gain chiffré avant observation.

### livrable

Un document court de décision comprenant :

- la situation de travail observée ;
- les tâches réellement concernées ;
- les irritants et points de friction ;
- les informations manipulées ;
- le tri : automatiser, assister, garder humain ;
- les risques : erreurs, surcharge, double contrôle, perte de repères, données instables ;
- un score d’opportunité et de vigilance ;
- une recommandation claire.

Conclusion possible :

- tester l’IA sur un cas limité ;
- recadrer le cas avant tout test ;
- ne pas lancer d’IA sur ce périmètre.

### déroulé

1. cadrage court — [30 à 60 min]
2. analyse du travail réel — entretiens courts, observation ou revue de cas/documents
3. synthèse — document de décision, pas rapport long
4. restitution — [60 à 90 min] avec les personnes clés

### conditions

Durée estimée : [à compléter]
Charge d’intervention : [x jours]
Prix : [à compléter] € HT
Modalités : [présentiel / distanciel / mixte]
Période souhaitée : [à compléter]
Validité : [30 jours]
Documents ou accès nécessaires : [à compléter]

### ce qui n’est pas inclus

- choix d’un logiciel ;
- paramétrage ou développement d’un outil IA ou agent ;
- cahier des charges technique complet ;
- formation générale à l’IA ;
- audit global de l’organisation ;
- promesse de gain financier chiffré sans mesure préalable.

### prochaine étape

Pour démarrer : valider le périmètre, confirmer les personnes à rencontrer, fixer les dates, valider les conditions.
